# Post-launch Monitoring

Monitor:

- `/api/health`
- `/api/readiness`
- service request submissions
- support handoffs
- quote requests
- admin auth failures
- technician route guard failures
- Simply Accounting export workflow events
- notification dispatch failures

Keep finance/export and service-role key logs private.


## Additional monitoring

Track auth bridge failures, notification dispatch failures, storage signing failures, and audit sink write failures.
