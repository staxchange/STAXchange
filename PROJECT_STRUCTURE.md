# STAX Product Structure

## Core product areas

- Exchange = discovery feed
- Market = search + listings
- Vault = owned items
- Deal Room = buy / trade / hybrid transaction flow
- Trade Board = recent completed public trades / transactions feed
- Break Room = live event rooms

## Launch build rules

- Keep launch UX simple
- Keep code modular
- Capture useful data now
- Do not turn on advanced intelligence yet
- Use feature flags for later systems

## Exchange rules

- Stack or Pass happens in the Exchange
- Right swipe = Stack
- Left swipe = Pass
- Stacked items are acknowledged as Tracked

## Future intelligence to preserve room for

- recommendation logic
- recency bias
- social weighting
- market insights
- consensus / judging later
- negotiation intelligence later
